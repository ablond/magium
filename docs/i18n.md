# I18n

## Goal

Prepare French and future languages without changing the game engine.

The V1 narrative uses `en` as the complete fallback. All of Book 1 (`ch1`
through `ch11b`), its achievements, and stats are also available in `fr`. The
UI shell is available in `en` and `fr`.

The model separates:

- logic graph;
- displayed narrative messages;
- UI messages.

## Principle

Logic must never be translated.

Do not translate or modify:

- scene IDs;
- targets;
- variables;
- conditions;
- assignments;
- special values;
- achievement variables;
- logical block order.

Translate only:

- paragraphs;
- choices;
- achievement titles/captions;
- stat labels;
- UI shell.

## Current Format

English locale and French Book 1 coverage:

```text
content/canonical/v1/locales/en/<chapterId>.json
content/canonical/v1/locales/en/achievements.json
content/canonical/v1/locales/en/stats.json
content/canonical/v1/locales/en/ui.json
content/canonical/v1/locales/fr/ch1.json
content/canonical/v1/locales/fr/ch2.json
content/canonical/v1/locales/fr/ch3.json
content/canonical/v1/locales/fr/ch4.json
content/canonical/v1/locales/fr/ch5.json
...
content/canonical/v1/locales/fr/ch11b.json
content/canonical/v1/locales/fr/achievements.json
content/canonical/v1/locales/fr/stats.json
content/canonical/v1/locales/fr/ui.json
```

Shape:

```json
{
  "locale": "en",
  "chapterId": "ch1",
  "messages": {
    "ch1.Ch1_Intro1.p1": "They say there is...",
    "ch1.Ch1_Intro1.c1": "Excited"
  }
}
```

The story graph references only `messageId`.

## Editable Sources

UI files come from:

```text
content/ui-locales/en.json
content/ui-locales/fr.json
```

They are packed at runtime under `locales/en/ui` and `locales/fr/ui`.
`index.uiLocales` lists available UI locales.

Translated narrative files come from:

```text
content/story-locales/en/stats.json
content/story-locales/fr/ch1.json
content/story-locales/fr/ch2.json
content/story-locales/fr/ch3.json
content/story-locales/fr/ch4.json
content/story-locales/fr/ch5.json
...
content/story-locales/fr/ch11b.json
content/story-locales/fr/achievements.json
content/story-locales/fr/stats.json
```

They are copied to canonical content and then packed under
`locales/<locale>/<bundle>`. `index.storyLocales` lists available story
languages.

## Global Language And Fallback

The FR/EN language setting is global:

- `settings.uiLocale` drives the interface shell;
- `settings.locale` and `GameState.locale` drive story, achievements, and stats;
- changing language reloads text packs but does not reset scene, variables, achievements, history, or `historyDigest`.

The runtime fallback remains `en`. If a chapter does not exist in the selected
locale, that chapter is displayed in English while translated interface, stats,
and achievements that are available remain in French.

## Adding A Language

Expected future procedure:

1. generate a locale skeleton from `en`;
2. keep exactly the same `messageId` keys;
3. fill translated values;
4. validate coverage;
5. generate runtime packs for the locale.

Target source path:

```text
content/story-locales/fr/<chapterId>.json
```

Generated canonical path:

```text
content/canonical/v1/locales/fr/<chapterId>.json
```

Generated runtime pack:

```text
src/generated/packs/locales__fr__<chapterId>.ts
```

## Fallback

The runtime falls back to `en` if a locale pack does not exist.

For a real future locale, add strict validation:

- every `en` key must exist in the locale;
- no extra keys unless an explicit convention allows them;
- coverage report by chapter;
- stats translated with the same keys as `en/stats.json`;
- achievements translated for every chapter covered by the locale, with valid `title` and `caption` keys against `en`;
- test that language switching preserves player state.

## Translation Export

Recommended exchange formats for future work:

- flat JSON for fast iteration;
- XLIFF for a more professional translation workflow.

The runtime format must remain generated packs, not XLIFF.

## Public Contributions

The PWA may show a correction form on a visible paragraph or choice. For
narrative text, the form targets the displayed paragraph within the `messageId`
through `segmentIndex`; it is not meant to rewrite an entire multi-paragraph
block. This form does not modify translation files and does not create a PR
directly.

Expected flow:

1. the reader submits an anonymous-by-default correction;
2. the payload sends hidden technical IDs and current/source text hashes, limited to the segment for paragraph corrections;
3. `services/translation-api` stores the proposal;
4. a maintainer accepts, rejects, or edits a final version;
5. accepted proposals are grouped in a changeset;
6. GitHub Actions applies the changeset to `content/story-locales/<locale>/<chapter>.json`, replacing only the target segment for a paragraph correction;
7. `pnpm content:all`, `pnpm check`, `pnpm test`, and `pnpm build` must pass before the PR.

A proposal made stale by an already-merged correction is detected through
`currentTextHash` and must be marked `stale`, not forced. For a paragraph, only
the target segment hash is compared: a neighboring paragraph change in the same
`messageId` is not enough to stale the proposal.

The credit pseudonym is optional and moderated. Follow-up email is optional,
confirmed by link, and the consent can be reused for one year in the same
browser through a local token. The API keeps only HMAC/hash proof of consent;
raw email remains limited to pending proposal or changeset contacts, then is
deleted after grouped rejection, stale, or publication notification.

## UI Shell

UI labels are in source bundles:

```text
content/ui-locales/en.json
content/ui-locales/fr.json
```

They are generated and loaded as runtime packs:

- `locales/<locale>/ui`
- fallback `locales/en/ui`

This covers:

- Saves;
- Stats;
- post-choice stat check results;
- Achievements;
- Settings;
- About;
- error notices;
- export/import labels.

`content:validate` checks that UI locales have exactly the same keys as `en`
and that matching packs exist.
