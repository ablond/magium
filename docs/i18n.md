# I18n

## Objectif

Preparer la traduction en francais puis dans d'autres langues sans changer le moteur de jeu.

La V1 utilise seulement `en`, mais le modele est deja separe :

- graphe logique ;
- messages affiches.

## Principe

La logique ne doit jamais etre traduite.

Ne pas traduire ou modifier :

- scene IDs ;
- targets ;
- variables ;
- conditions ;
- assignments ;
- special values ;
- achievement variables ;
- ordre logique des blocs.

Traduire seulement :

- paragraphes ;
- choix ;
- titres/captions d'achievements ;
- UI shell.

## Format Actuel

Locale anglaise :

```text
content/canonical/v1/locales/en/<chapterId>.json
content/canonical/v1/locales/en/achievements.json
```

Forme :

```json
{
  "locale": "en",
  "chapterId": "ch1",
  "messages": {
    "ch1.Ch1_Intro1.p1": "They say there is...",
    "ch1.Ch1_Intro1.c1": "Excited"
  }
}
```

Le graphe reference uniquement les `messageId`.

## Ajouter Une Langue

Future procedure attendue :

1. generer un squelette de locale depuis `en` ;
2. conserver exactement les memes `messageId` ;
3. remplir les valeurs traduites ;
4. valider la couverture ;
5. generer les runtime packs de la locale.

Chemin cible :

```text
content/canonical/v1/locales/fr/<chapterId>.json
```

Puis pack runtime :

```text
src/generated/packs/locales__fr__<chapterId>.ts
```

## Fallback

Le runtime actuel tombe sur `en` si un pack locale n'existe pas.

Pour une vraie locale future, ajouter une validation stricte :

- toutes les clés `en` doivent exister dans la locale ;
- pas de clés en plus sauf convention explicite ;
- rapport de couverture par chapitre ;
- test que le changement de langue conserve l'etat joueur.

## Export Traduction

Format d'echange recommande pour la suite :

- JSON plat pour iteration rapide ;
- XLIFF pour workflow traduction plus professionnel.

Le format runtime doit rester les packs generes, pas XLIFF.

## UI Shell

Actuellement, plusieurs libelles UI sont dans `App.svelte`.

Avant de commencer la traduction, sortir ces libelles dans un bundle UI :

```text
content/canonical/v1/locales/en/ui.json
```

Puis charger :

- `locales/<locale>/ui`
- fallback `locales/en/ui`

Cela concerne :

- Saves ;
- Stats ;
- Achievements ;
- Settings ;
- About ;
- notices d'erreur ;
- labels export/import.
