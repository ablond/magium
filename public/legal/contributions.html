Contributions, Personal Data, and Moderation

The full legal notices for the public instance must be completed by its operator before the contribution form is opened.

Contributing Without An Account

Translation corrections can be submitted without an account, without a pseudonym, and without an email address. The service receives only the proposed correction, the technical context needed to find the passage, and an anti-bot signal.

Optional Email

Email is optional. If provided, it is used only to confirm the address and send follow-up notifications for the related proposals. If several proposals are processed in the same batch for the same confirmed address, only one summary email is sent. The address is not displayed publicly and is deleted after the proposal is rejected, marked stale, or published.

Optional Pseudonym

The pseudonym is optional. It is used only to credit an accepted contribution when the credit checkbox is selected. It may be moderated or hidden if it is illegal, violent, hateful, sexually explicit, child-sexual-abuse material, impersonates someone, exposes personal data, or is clearly inappropriate.

Validation And License

An accepted proposal may be editorially adjusted by maintainers before integration. If retained, it is integrated through a GitHub pull request validated by the repository checks. Project code, documentation, and configuration are MIT-licensed; Magium story/data and derived narrative translations remain under the upstream CC BY 4.0 notice.

Anti-Bot

The form uses a technical anti-bot protection configured by the instance. The documented default is Cloudflare Turnstile in non-interactive/invisible mode.
