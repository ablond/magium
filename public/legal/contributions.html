Contributions, données personnelles et modération

Les mentions légales complètes de l'instance publique doivent être complétées par son éditeur avant l'ouverture du formulaire de contribution.

Contribuer sans compte

Les corrections de traduction peuvent être envoyées sans compte, sans pseudo et sans email. Le service reçoit uniquement la correction proposée, le contexte technique nécessaire pour retrouver le passage, et un signal anti-bot.

Email optionnel

L'email est facultatif. S'il est renseigné, il sert uniquement à confirmer puis notifier le suivi des propositions concernées. Si plusieurs propositions sont traitées dans le même lot pour la même adresse confirmée, un seul email récapitulatif est envoyé. L'adresse n'est pas affichée publiquement et elle est supprimée après notification de refus, d'obsolescence ou de publication.

Pseudo optionnel

Le pseudo est facultatif. Il sert uniquement à citer une contribution retenue si la case de crédit est cochée. Il peut être modéré ou masqué s'il est illégal, violent, haineux, sexuel explicite, pédopornographique, usurpe une identité, divulgue des données personnelles ou est manifestement inadapté.

Validation et licence

Une proposition acceptée peut être modifiée éditorialement par les mainteneurs avant intégration. Si elle est retenue, elle est intégrée au projet sous la licence de l'adaptation et passe par une pull request GitHub validée par les contrôles du dépôt.

Anti-bot

Le formulaire utilise une protection anti-bot technique, configurée par l'instance. Le choix par défaut documenté est Cloudflare Turnstile en mode non interactif/invisible.
